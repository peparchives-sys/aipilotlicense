§ 4.1 8 min read Doctrine × 1

Three-stage pipeline.

Learning objective — given a real deliverable, decompose the work into discrete stages with inspectable handoffs and explicit review gates.

An amateur asks the model to write the report. A pilot does not. A pilot identifies the work as a pipeline — a sequence of bounded operations, each one inspectable, each one rejectable, each one with a clear handoff into the next.

The temptation, when AI is involved, is to collapse all three stages into a single prompt. The output looks fluent. The output looks finished. The output is, almost always, structurally wrong somewhere — and because it was generated in one breath, you cannot see where.

Doctrine 4.1

The work is the workflow. A bad workflow with a great model produces a polished failure. A great workflow with a passable model produces dependable work.

Stage one — structure extraction

You do not begin with prose. You begin with structure. Hand the model the source materials and ask it to produce only the architecture: section list, claim list, evidence map. No drafting. The output is rejected and re-issued until the architecture is sound.

Stage two — drafting against the structure

Only when the architecture is approved does drafting begin — and the prompt now references the locked structure as a binding constraint, not a suggestion. The model fills in. It does not invent shape.

Stage three — editorial pass

The third stage is not "polish." The third stage is adversarial. You hand the draft back to the model with a different role and a different objective: find what is weak, find what is unsupported, find what overclaims. Then you, the pilot, decide what to act on.

You are producing a board-level acquisition memo. You have hand-collected source documents but no prior template. You decide to use AI as the execution engine. Which of the following best reflects CAIP doctrine for structuring the work?
  • A
    Hand the source documents to the model and ask it to write the memo in a single prompt, then revise as needed.
  • B
    Decompose the work into a pipeline: (i) extract structure, (ii) draft against the locked structure, (iii) adversarial editorial pass — with explicit review gates between stages.
  • C
    Run three independent prompts, ask the model to pick its best output, and forward that one to the board.
  • D
    Refuse the assignment until you have a hand-built template that the model can populate verbatim.
